## 알려진 한계 (해결 예정)

### 1. LLM 성공률 ~38%
60 LLM 호출 중 37건 JSON 파싱 실패 → 휴리스틱 폴백.  
원인: Haiku가 한글 문자열에서 가끔 JSON 깨뜨림 (이스케이프 이슈).  
**해결책 후보**: response_format 강제 / structured outputs / 재시도 1회 / Sonnet 폴백.

### 2. 슬롯 데이터 풍부함
임포트 시 평일 1슬롯 선택 = 5일 모두에 그 슬롯 추가하는 식이라 슬롯 매우 많음.  
실제 운영 시 참가자가 날짜별로 콕 집어 선택하면 더 까다로워짐.

### 3. 알고리즘 학습 인터뷰 (다음 단계)
지금은 휴리스틱 + LLM "잘 맞아 보임" 기반.  
**호스트의 과거 매칭 노하우를 인터뷰** → `docs/matching-heuristics.md` 작성 → Claude system prompt에 주입 → 더 정교한 매칭.

### 4. 남자 오픈채팅 URL 미입력 처리
현재는 매칭에 안 쓰이지만, 매칭 후 SMS 발송 시 URL 없으면 여자한테 보낼 게 없음.
