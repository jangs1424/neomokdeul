## 📨 1. 신청 관리

### Q1.1 기수별 뷰 방식

여러 기수가 동시 모집중일 수 있어. 어떻게 분리해서 볼까?

- [ ] **탭 형태** — 상단에 `전체 | 2026 5월 기수 | 2026 6월 기수 | …` 탭. 클릭하면 그 기수만
- [ ] **드롭다운** — 상단에 기수 선택 드롭다운 + "전체 보기" 옵션
- [ ] **URL 파라미터** — `/applications?cohort=xxx` (공유 링크 편함)
- [ ] 위 조합 (예: 탭 + URL 파라미터)

**답:** 

---

### Q1.2 상단 카운트 위젯

페이지 상단에 뭐가 보여야 해? (복수 선택 가능)

- [ ] 대기 건수 (전체)
- [ ] 대기 건수 (남/여 분리)
- [ ] 승인 건수 (남/여 분리)
- [ ] 반려 건수
- [ ] 결제 완료 건수
- [ ] 정원 대비 남은 자리 (예: "남 5/15 · 여 8/15")
- [ ] 기수별 요약 여러 개를 카드로 나열
- [ ] 다른 것:

**답:** 

---

### Q1.3 음성 자소 — 리스트에서 어떻게 들을까?

- [ ] 리스트 행에 작은 **▶ 재생 버튼** 바로 표시 (인라인 재생)
- [ ] 행 클릭 → 상세 패널 열리고 거기서 재생
- [ ] 별도 팝업(모달) 열려서 재생
- [ ] 재생 속도 조절 필요? (0.5x / 1x / 1.5x)
- [ ] 재생 이력 기록 필요? ("이 건은 들어봄" 표시)

**답:** 

---

### Q1.4 얼굴 사진 — 어떻게 볼까?

- [ ] 리스트 행에 **작은 썸네일**(40~60px) 노출
- [ ] 썸네일 클릭 → 원본 크게
- [ ] 썸네일 없이, 행 클릭 → 상세에서 보여줌
- [ ] 사진 비공개 기수도 있어야 함? (예: 테마별로)

**답:** 

---

### Q1.5 승인 버튼 누르면 뭐가 일어나야 해?

- [ ] 상태만 `approved`로 변경. 문자는 별도 /messages 에서 발송
- [ ] 즉시 문자 발송 (실수 방지 위해 확인 다이얼로그 1번)
- [ ] "승인 → 바로 문자 발송" / "승인만" 두 버튼 따로
- [ ] 승인 시 승인자에게 **자동 메모 템플릿** 적기? (예: "음성 OK, 동기 good")

**답:** 

---

### Q1.6 반려 버튼

- [ ] 반려 시 반드시 **반려 사유 메모** 입력 (팝업으로)
- [ ] 사유 선택 옵션 미리 만들어둠 (예: "성별 정원 초과" / "음성 자소 미흡" / "MBTI 중복" / "기타")
- [ ] 반려는 그냥 클릭 (사유 선택)
- [ ] 반려 후 되돌릴 수 있어야 함 (승인으로 변경 가능)

**답:** 

---

- [ ] 이름 · 상태 · 모집기간 · 프로그램기간

- [ ] 다른 것:

**답:** 

---

### Q2.2 기수 상태 전이

기수는 `draft → recruiting → closed → running → completed` 단계.

- [ ] 상태 변경은 호스트가 수동으로 (버튼 클릭)
- [ ] 자동으로 전환 (예: `apply_closes_at` 지나면 자동 closed)
- [ ] 자동 + 수동 override 가능

**답:** 

---

### Q2.3 기수 복제

- [ ] 이전 기수 설정(폼 문구·SMS 템플릿 등)을 복사해서 새 기수 빠르게 시작 — 필요
- [ ] 필요 없음, 매번 새로 입력해도 OK

**답:** 

---

### Q2.4 completed(완료) 기수 데이터

- [ ] 완료 기수는 수정 잠금 (편집 불가)
- [ ] 완료 후에도 편집 가능
- [ ] 완료 기수는 별도 탭에서 보기

## 📱 3. 문자 발송

### Q3.1 발송 대기 목록 기준

- [ ] 승인됐는데 `payment_link_sent_at` 비어있는 건만 대기 목록
- [ ] 호스트가 수동으로 "발송 대기" 큐에 담기
- [ ] 기수별로 분리해서 보기

**답:** 

---

### Q3.2 발송 방식

- [ ] 한 번에 여러 명 선택 → 일괄 발송 (기본)
- [ ] 한 명씩 개별 발송
- [ ] 둘 다 지원

**답:** 

---

### Q3.3 미리보기

- [ ] 발송 직전 한 명의 미리보기 보여줌 (대표 1건)
- [ ] 각 명별 미리보기 전부 보여줌 (변수 치환된 실제 문자)
- [ ] 발송 전에 "발송할 건수 + 샘플 1건" 만 보여주고 확인

**답:** 

---

### Q3.4 발송 직전 수정

- [ ] 템플릿만 수정 가능 (기수 전체)
- [ ] 개별 건별 문자 수정 가능 (예: A한테만 다른 문구)
- [ ] 수정 없이 템플릿 그대로만

**답:** 

---

### Q3.5 발송 실패 처리

- [ ] 실패 건 자동 재시도 (3회)
- [ ] 실패 건 별도 리스트 → 수동 재시도
- [ ] 일단 로그만 남기고 수동 처리

**답:** 

---

### Q3.6 발송 기록

- [ ] 누구한테 언제 뭘 보냈는지 영구 기록 (감사 로그)
- [ ] 발송된 문자 원문 저장
- [ ] 기록 없이 status만 기록

**답:**

## 🎯 4. 매칭 실행

### Q4.2 1차 + 2차 매칭 동시 생성?

스펙상: Day 1에 1차+2차 동시 생성, 이탈자만 교체.

- [ ] 스펙대로 1차 + 2차 동시 생성
- [ ] 1차 먼저, 2차는 Day 4 저녁에 따로
- [ ] 상황 봐서 결정 (매칭 실행 시 옵션)

**답:** 

---

### Q4.3 매칭 결과 검토 UI

- [ ] 남-여 쌍을 카드로 2열 배치, 각 카드에 궁합점수·매칭사유
- [ ] 테이블 형태로 쭉 나열
- [ ] 다른 것:

**답:**

### Q4.4 드래그로 교체

- [ ] 남 카드를 다른 여 카드로 드래그 → 즉시 교체

### Q4.5 exclusions (재참가자)

- [ ] 이전 기수 매칭 쌍 자동 제외 (전화번호 기반)
